Frequently asked questions

Does my Search Console data get uploaded to a server? +
No. The CSV is parsed entirely in your browser using JavaScript. Nothing is uploaded, stored, or sent anywhere — you can disconnect from the internet after the page loads and the tool still works.
Where do I export the file this tool needs? +
In Google Search Console, open Performance, set your date range, then click Export at the top and choose Excel (.xlsx). That single file contains a Queries tab and a Pages tab plus a few others — this tool reads both automatically and lets you switch between them. A plain Queries.csv or Pages.csv export also works.
Why do some patterns say they're disabled? +
A handful of patterns need a personalized input to work correctly, such as your brand name, competitor names, or domain. Fill in the optional personalization fields above the results to enable those patterns.
Are these the same regex patterns Search Console itself uses? +
Google Search Console's filter uses the RE2 (Golang) regex flavor, while this tool runs in JavaScript's regex engine. The patterns are functionally equivalent, but a couple were adapted slightly (for example, inline case-insensitive flags) so they compile correctly in the browser.
Can I test my own regex pattern instead of the built-in ones? +
Yes. The Custom Regex Tester tab lets you write any pattern, choose Query or Page as the field, choose matches or does-not-match, and see live results with impression and click totals.
