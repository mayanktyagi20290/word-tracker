Frequently asked questions

What does the regex explainer actually do?▾
It breaks your pattern into individual tokens — character classes, quantifiers, groups, anchors, escape sequences — and describes each one in plain English, so you can understand or debug a pattern without memorizing regex syntax.
What regex flavor does this tool use?▾
JavaScript's native regular expression engine (ECMAScript), the same one used in browsers and Node.js. Syntax mostly overlaps with PCRE used by PHP, Python, and other languages, but some advanced features may differ.
Why does my pattern only match once even though the text repeats?▾
Without the global (g) flag, JavaScript regex only returns the first match. Enable the global flag to find every occurrence in the test string.
How do I use capture groups in the replace field?▾
Reference a numbered group with $1, $2, and so on, or a named group with $<name>. $& refers to the whole match, and $$ inserts a literal dollar sign.
Is my regex pattern or test text sent anywhere?▾
No. Matching, replacing, and explaining all run locally in your browser using JavaScript's built-in regex engine. Nothing is uploaded.
