Frequently asked questions

What is a sitemap index file and does this tool support it?
A sitemap index file is a sitemap that points to other sitemaps instead of pages — common on large sites that split URLs into folders like sitemap-posts.xml and sitemap-products.xml. This tool automatically detects index sitemaps and recursively crawls every child sitemap, combining all URLs into one result.
How many URLs can I extract from a sitemap?
There is no hard cap. The tool will keep crawling nested sitemaps and collecting URLs until the sitemap tree is fully extracted, or until the limit you set in Max URLs is reached.
Can I filter URLs by folder or path?
Yes. Use the Include Paths box to only keep URLs containing certain folder names or segments, and the Exclude Paths box to remove URLs matching unwanted paths — useful for isolating just /blog/ or excluding /tag/ pages.
Does this tool show lastmod, priority, and changefreq?
Yes, when the sitemap includes that metadata it is shown alongside each URL in the results table, and is included in the CSV export.
Is my data sent to a third-party server?
The sitemap is fetched through a lightweight same-site proxy to avoid browser CORS restrictions, parsed entirely in your browser, and never stored or logged.
